LapX — Best Path Forward

Goal: reach a durable, hardware-driven race with the fewest moving parts, then add production UI and optional administration only when the preceding tracer works.

P0 requiredP1 operating valueP2 optional productP3 defer

Priority changes from the local plan

  • Finish rules move before Fehlstart, Rennpause, and chaos: first prove one complete race.
  • SQLite/CLI remains before HTTP: durability has one command path before transport.
  • Hardware follows the static debug tracer; production SvelteKit follows working hardware.
  • Manual tournaments precede Elo and generated tournaments. Random generation can ship without Elo balancing.
  • P0 ends with hardware plus API readiness; the first browsable milestone is the P1 Debug-Bedienung at Delivery #13.

Slice numbers preserve traceability to the original local plan. Delivery numbers are the recommended implementation order.
