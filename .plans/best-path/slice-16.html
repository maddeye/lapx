Slice 16 — GPIO Messereignisse

P0Delivery #16Hardware enablement

Use the Raspberry Pi library’s edge callbacks so timestamps are captured near the signal.

Best path forward

  1. Use rppal GPIO edge callbacks; fall back to polling only if a measured Pi check shows callbacks cannot meet the required timestamp resolution.
  2. Add one process-level hardware config for BCM pins, active edge, and pull mode; keep it separate from per-race RaceConfig and preserve Mindestrundenzeit as the lap filter.
  3. Capture the monotonic timestamp first in the callback, then enqueue the lane edge for the Slice 15 adapter; never perform SQLite work in the callback.

Verification

  • cargo test --features gpio
  • On Pi: each configured input commits the correct lane event
  • Measure repeated trigger capture and confirm ≤1 ms timestamp resolution

Keep it simple: not in this slice

  • A second debounce model
  • Kernel module
  • Hard-coded pin map
