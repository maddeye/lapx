Slice 1 — Race configuration shell

P0Delivery #1Foundation

Define the smallest valid RaceEngine boundary before adding race behavior.

Best path forward

  1. Create one domain module containing RaceConfig, Command, Event, RaceState, and validation.
  2. Accept 1–4 lanes; reject invalid durations, targets, lanes, and commands that do not apply to the current phase.
  3. Expose RaceEngine::handle(command) -> Result<Vec<Event>, DomainError>; do not add persistence or transport yet.

Verification

  • HTTP or UI
  • Hardware adapters
  • A framework or separate DTO layer
