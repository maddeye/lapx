Slice 18 — Hardware debug page

P1Delivery #18Hardware enablement

Expose one read-only commissioning snapshot; reuse the existing debug-page approach.

Best path forward

  1. GET /api/hardware returns configured pins, current input/output levels, and latest raw edge per lane.
  2. Serve a local-only static /hardware page that polls the snapshot; no new live transport and no public route.
  3. Keep a single latest edge per lane instead of building a history store.

Verification

  • cargo test hardware_page_loads_snapshot
  • On Pi: input and relay changes appear with correct lane/pin mapping

Keep it simple: not in this slice

  • Oscilloscope UI
  • Remote GPIO writes
  • Persistent raw signal history
