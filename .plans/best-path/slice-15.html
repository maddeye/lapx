Slice 15 — TimingSource and PowerOutput seams

P0Delivery #15Hardware enablement

Add two narrow adapter boundaries only when real hardware is next.

Best path forward

  1. TimingSource captures a monotonic Instant; the adapter immediately converts it to the integer protocol milliseconds defined in Slice 3. On restart, seed the new monotonic mapping after the last committed protocol timestamp.
  2. Extend RaceRuntime so every live command source—HTTP, due-event task, or TimingSource—uses the same commit path; after commit it broadcasts state and synchronizes PowerOutput. EventStore and RaceEngine stay hardware-free.
  3. Record each captured edge once and retry only transient busy-store failures before processing later ingress, preserving its original timestamp and order. Never retry a possible post-commit power failure.
  4. Provide simulation implementations for those adapters.
  5. Keep traits and adapters outside RaceEngine; avoid a generic hardware framework.
