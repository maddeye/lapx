Slice 9 — SQLite Rennprotokoll and lapxctl

P0Delivery #6First durable race

Persist the first complete race transactionally before adding optional control rules.

Best path forward

  1. Create one append-only race_events table keyed by race and sequence; store an explicit event type and schema version beside the JSON payload.
  2. Use BEGIN IMMEDIATE as the single-writer boundary: load/replay, handle one command, append all resulting events, and commit; only then expose new state.
  3. Use lapxctl <command> --json <file>, with - for stdin, and JSON state on stdout; no interactive shell.
  4. Enable WAL and synchronous=FULL as required by ADR 0001.

Verification

  • cargo test store_cli sqlite_store_replays_committed_events
  • Close and reopen a temporary database; replay must match pre-close state
  • Two concurrent writers serialize without lost events or duplicate sequence numbers

Keep it simple: not in this slice

  • ORM
  • Snapshots or migrations framework before a second schema version
  • Interactive TUI

Inspiration PRs

  • Trading Bot append-only capital events
  • Pitchfork CLI JSON output
