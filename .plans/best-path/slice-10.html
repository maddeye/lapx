Slice 10 — Local HTTP state endpoint

P0Delivery #10Local integration

Add only a local read endpoint over the durable projection.

Best path forward

  1. Use one small Rust HTTP server and shared EventStore.
  2. GET /api/state loads and projects current RaceState at request time; add caching only if measured replay latency exceeds 100 ms.
  3. Bind to loopback by default. Run one server-owned due-event task that sleeps until the earliest projected due timestamp, then applies AdvanceRace; committed state changes wake it to recalculate. Starts, finishes, restarts, and power restoration must not depend on a browser.

Verification

  • cargo test http_state_returns_json runtime_materializes_due_event
  • An official start becomes committed after its due time with no client request

Keep it simple: not in this slice

  • HTTP command routes
  • SSE
  • CORS, auth, caching, or public bind
