Slice 3 — Official start as a due event

P0Delivery #3Foundation

Represent official start explicitly so replay never depends on a wall clock.

Best path forward

  1. Represent protocol time as integer milliseconds supplied to RaceEngine; store Starting { due_at } in that timeline, never as a wall-clock read inside the engine.
  2. AdvanceRace(to) emits OfficialStart only when due_at is reached; normal live commands first materialize any earlier due event.
  3. Reject command timestamps older than the latest committed protocol timestamp; live adapters derive increasing values from a monotonic clock.

Verification

  • cargo test start_sequence replay_at commands_before_last
  • A command after the due time commits OfficialStart first, at its due timestamp
  • Replay ending before the due timestamp remains in Startsequenz

Keep it simple: not in this slice

  • Background timer inside RaceEngine
  • Countdown UI
  • General-purpose scheduler
